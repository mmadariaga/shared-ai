## Why

The opencode budget skills (`budget-explorer`, `budget-executor`, `budget-subagent`) lacked documentation explaining the rationale for delegating work to subagents. Their claude counterparts already carried a `## Cost model` section with a **Why delegate** block covering cost efficiency and context hygiene. This gap meant agents loading the opencode skills had no documented justification for the delegation pattern.

## What Changes

- `skills/opencode/budget-explorer/SKILL.md` — added `## Cost model` section after `## Model resolution`, before `## Output contract`
- `skills/opencode/budget-executor/SKILL.md` — added `## Cost model` section after `## OpenCode Binding`
- `skills/opencode/budget-subagent/SKILL.md` — added `## Cost model` section after `## OpenCode Binding`

## Capabilities

### New Capabilities

- `opencode-budget-cost-model-docs`: Each opencode budget skill now documents the cost and context-hygiene rationale for subagent delegation, adapted to opencode's model-resolution mechanism (`agent.<keyword>.model` in `opencode.jsonc`).
