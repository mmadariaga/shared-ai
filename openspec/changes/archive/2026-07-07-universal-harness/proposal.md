> **⚠ POST-HOC RECORD** — This proposal was backfilled after implementation. It describes a decision already made, not one being proposed.

## Why

The shared-AI pipeline was built around two harnesses (Claude Code and opencode), and several of its rules, documentation paths, and skill bindings were coupled to that two-harness assumption. Adding GitHub Copilot as a third supported harness required generalizing those rules so they hold for any number of harnesses — and naming Copilot explicitly in the places where the generalization still needs a concrete third instance (cost-discipline subagent tiers, project-local skill/command paths, skill compatibility metadata, prerequisite-check fetch resolution).

## What Changes

- `AGENTS.md` — repo structure table extended with `skills/copilot/`, `agents/copilot/`, `.github/skills/`, `.github/prompts/`, `~/.copilot/agents/`; prerequisite-check paragraph generalized to mention the Copilot SAI folder; Cost Discipline paragraph names three subagent bindings; project-local commands and skill-fetches lists include Copilot equivalents; typo fix `ai-*` → `sai-*` in two places.
- `GLOSSARY.md` (new) — defines the **Backfilled Change** term, its relationships, and an example dialogue.
- `README.md` — three `budget-*` skill rows extended to include the Copilot `GPT-5 mini (copilot)` tier and the `agents/copilot/<name>.agent.md` binding; three narrative paragraphs generalized from "Claude Code and Opencode" to "Claude Code, opencode, and GitHub Copilot"; the npx installer description updated to list three targets.
- `sai/instructions/apply.md` — `## Step-Execution Subagent Dispatch` rewritten to dispatch via the per-harness `budget-subagent` skill binding (write-capable, cheap tier) instead of the hardcoded `subagent_type: general-purpose` + "no `model:` param" carve-out tied to `docs/adr/0017`.
- `sai/instructions/commit-rules.md` — three literal mentions of `"Generated with Claude Code"` replaced with the broader phrase `AI-generated attribution trailers`. The forbidden-trailer ban itself is unchanged.

## Capabilities

### New Capabilities

- `skill-copilot-compatibility`: A skill's YAML `compatibility:` field SHALL declare `copilot` alongside `claude` and `opencode` whenever the skill works on all three harnesses — enabling the harness-loader to gate skill installation per harness.

### Modified Capabilities

- `harness-universality`: AGENTS.md and the harness-skill/agent directory layout were extended for Copilot — repo structure now lists `skills/copilot/`, `agents/copilot/`, `.github/skills/`, `.github/prompts/`, and `~/.copilot/agents/`; prerequisite-check and Cost Discipline paragraphs name three harnesses; skill-fetches and project-local-commands lists include Copilot paths.
- `apply`: The Step-execution subagent dispatch mechanism switched from a hardcoded `subagent_type: general-purpose` + "no `model:` param" override to the per-harness `budget-subagent` skill binding (write-capable, cheap tier). The previous "same model as coordinator" carve-out was removed; the docs/adr/0017 reference was dropped.
- `commit-rules`: The forbidden-trailer category was generalized from the literal `"Generated with Claude Code"` to the broader `AI-generated attribution trailers`. Behavior is unchanged — only the identifier used in the rule text.
