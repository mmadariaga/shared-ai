## Why

The re-run guard rule — which requires the existing `implementation.md` checkbox scan to run inside a subagent — was not being enforced at runtime. The prior instruction embedded the directive as a blockquote hint (`> **Subagent:** ...`) buried inside a sub-step (Step 1b), making it easy for the agent to process the step inline instead. Rewriting it as a top-level Step 1 with a bold mandatory directive and a fully self-contained subagent prompt ensures the rule is applied. Completion messages in sai-5 through sai-8 were also simplified by removing the redundant path suffix.
