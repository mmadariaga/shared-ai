> **⚠ POST-HOC RECORD** — This proposal was backfilled after implementation. It describes a decision already made, not one being proposed.

## Why

The SAI pipeline had two error-prone behaviors: (1) the STOP & COMMIT marker in `implementation.md` was ambiguous — agents sometimes committed silently or skipped the gate entirely, and (2) the Copilot harness instruction file (`sai/instructions/harness/copilot.md`) duplicated rules that already belonged in universal locations (`prereqs.md`, `remember.md`), causing drift and stale fetch directives in 12 wrapper files.

## What Changes

- Deleted `sai/instructions/harness/copilot.md`; moved its OpenSpec path rules into `sai/instructions/prereqs.md` and its checkbox discipline rule into `sai/instructions/remember.md`.
- Removed `Fetch @sai/instructions/harness/copilot.md` from all 12 `commands/copilot/*.prompt.md` wrappers.
- Added a 4-step STOP & COMMIT checklist to `sai/instructions/apply.md` that requires explicit user authorization before every `git commit`.
- Added an explicit per-invocation authorization rule to `sai/instructions/commit-rules.md`.
- Updated `openspec/schemas/sai-workflow/templates/implementation.md` STOP & COMMIT sections to reference the new checklist instead of inline git commands.
- Updated 4 existing specs (`apply`, `copilot-checkbox-discipline`, `copilot-harness-instructions`, `install-instructions-recursive`) to reflect the changes.
- Removed 2 tests that asserted `harness/copilot.md` existence after install.
- Updated `AGENTS.md` description of `prereqs.md` to mention path resolution guidance.

## Capabilities

### New Capabilities

`stop-commit-checklist`: A mandatory 4-step sequence (propose message, ask explicitly, wait, act on response) that every STOP & COMMIT marker in `implementation.md` must follow.

`commit-auth-gate`: An explicit per-invocation authorization rule in `commit-rules.md` requiring the agent to ask before every `git commit` and to describe staged changes if the user declines.

### Modified Capabilities

`copilot-harness-removal`: The Copilot harness instruction file is retired; its path rules and checkbox rule now live in universal locations (`prereqs.md`, `remember.md`), eliminating a source of fetch-drift across 12 wrappers.
