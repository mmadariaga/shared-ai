## Why

The existing branch selection during `/sai-3-implement` only offered a default `{feature-name}` or reuse of the current branch. Users needed the ability to specify a backlog-linked branch name (e.g., `JIRA-123-feature-name`) as a first-class option rather than relying on free-text input that wasn't explicitly surfaced.

## What Changes

- Modified `sai/instructions/implement.md` Prerequisites section to present a structured branch selection prompt with three options:
  1. `{feature-name}` (derived from change name)
  2. Custom branch name (free input — e.g., backlog-linked name like `JIRA-123-feature-name`)
- Added instruction to create the selected branch from `main` if it does not already exist.

## Capabilities

### New Capabilities

- `custom-branch-name-option`: The branch selection prompt explicitly offers a custom branch name option with examples of backlog-linked naming patterns.
- `branch-creation-fallback`: If the selected branch does not exist, the agent creates it from `main` before proceeding with implementation.

### Modified Capabilities

- `sai-implement-branch-prompt`: Extended the existing branch prompt behavior from 2 options (default + reuse current) to 3 options (default + custom + create-if-missing).
