> **⚠ POST-HOC RECORD** — This proposal was backfilled after implementation. It describes a decision already made, not one being proposed.

## Why

Each SAI step (spec, design) performs codebase and external research to inform the artifacts it writes. That research was ephemeral — the next step started from zero, re-reading the same files or re-discovering the same URLs. Capturing the consulted sources directly inside the artifact eliminates that redundancy and creates an audit trail for decisions.

## What Changes

- `openspec/schemas/sai-workflow/schema.yaml` — Added `**Proposal Research Documentation**` instruction to the proposal artifact description, requiring agents to list files and URLs consulted during spec-phase research.
- `openspec/schemas/sai-workflow/templates/proposal.md` — Added `## Proposal Research Documentation` section with `### Local files` and `### External URLs` subsections.
- `openspec/schemas/sai-workflow/templates/tasks.md` — Added `## Required Documentation` section with `### Local files` and `### External URLs` subsections for design-phase research.
- `sai/commands/sai-1-spec.md` through `sai-8-accessibility.md`, `sai-backfill.md` — Added `## Completion` header wrapping existing MANDATORY STOP directives, standardizing command termination structure.
- `sai/instructions/design.md` — Bolded approval language; removed Completion section (now in command wrapper).
- `sai/instructions/performance.md`, `pr.md`, `review.md`, `security.md` — Removed outer `<TASK>` XML wrapper and flattened indentation to standard markdown.
- `sai/instructions/spec.propose.md` — Removed internal MANDATORY STOP (now owned by `sai-1-spec.md` wrapper).

## Capabilities

### New Capabilities

- `proposal-research-documentation`: proposal.md artifacts SHALL include a `## Proposal Research Documentation` section listing every local file and external URL consulted during spec-phase research.
- `tasks-research-documentation-template`: tasks.md template now includes the `## Required Documentation` section scaffold, ensuring design agents have a pre-formatted location to record design-phase sources.
- `command-completion-header`: sai command body files use a standardized `## Completion` section to wrap MANDATORY STOP directives.

### Modified Capabilities

- `sai-instruction-format`: Four instruction files (`performance.md`, `pr.md`, `review.md`, `security.md`) converted from `<TASK>`-wrapped format to flat markdown, normalizing the instruction file format across the suite.
- `spec-propose-stop-ownership`: MANDATORY STOP for spec generation now lives solely in the `sai-1-spec.md` command wrapper, not duplicated inside `spec.propose.md`.
