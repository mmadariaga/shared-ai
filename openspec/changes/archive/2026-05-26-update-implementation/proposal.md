> **⚠ POST-HOC RECORD** — This proposal was backfilled after implementation. It describes a decision already made, not one being proposed.

## Why

The `/sai-3-implement` command had no mechanism to incorporate findings from audit artifacts (review, security, performance, accessibility) into the implementation plan it generates. When these artifacts existed for a change, their actionable findings were silently ignored during `implementation.md` generation, requiring manual follow-up outside the workflow.

## What Changes

- `sai/instructions/implement.md` — Added an "Exception (audit artifacts)" block in Step 2 that instructs the agent to check for the existence of `review.md`, `security.md`, `performance.md`, and `accessibility.md` in the change directory, read each that exists, extract actionable findings, and append them as dedicated steps at the end of `implementation.md`.

## Capabilities

### New Capabilities

- `audit-artifact-ingestion`: `/sai-3-implement` reads existing audit artifacts for a change and appends their actionable findings as numbered implementation steps at the end of `implementation.md`.

### Modified Capabilities

None.
