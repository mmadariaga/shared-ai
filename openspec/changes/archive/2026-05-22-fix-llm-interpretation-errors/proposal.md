> **⚠ POST-HOC RECORD** — This proposal was backfilled after implementation. It describes a decision already made, not one being proposed.

## Why

LLM agents interpreting the sai-* command wrappers and instruction files produced incorrect or inconsistent behavior due to ambiguous completion messages, misplaced RUN blocks in instruction files, stale path references, and platform-prefixed skill names. This change standardizes completion phrasing, removes `$ARGUMENTS` run blocks from instruction files (owned by wrappers), corrects the backfill spec output path, normalizes budget skill names, and adds a re-run guard to the implement instruction.

## What Changes

- All `sai/commands/sai-*.md` completion messages standardized from "Once implementation is done" to "Once all artifacts are written your work is COMPLETE"
- `sai/instructions/security.md`, `review.md`, `performance.md` — removed trailing `## Run` / `$ARGUMENTS` blocks (these belong in command wrappers, not instruction files); fixed `<output_template>` indentation in security.md
- `sai/instructions/backfill.md` — corrected spec output path from `openspec/specs/` to `openspec/changes/{name}/specs/`
- `sai/instructions/implement.md` — added Step 1b re-run guard for detecting already-applied steps in existing `implementation.md`
- `skills/claude/budget-*/SKILL.md` and `skills/opencode/budget-*/SKILL.md` — normalized `name:` frontmatter from platform-prefixed to platform-agnostic slugs
- Scope reminders updated: `ai-3-apply` → `sai-4-apply` across security, review, performance, accessibility instructions
- `INSTALL.claude.md` and archived proposal — corrected command reference to `sai-4-apply`

## Capabilities

### New Capabilities

- `command-completion-standardization`: All sai-* command body files use consistent "Once all artifacts are written your work is COMPLETE" completion phrasing in their `## Completion` section
- `instruction-run-block-removal`: Instruction files under `sai/instructions/` no longer contain `## Run` / `$ARGUMENTS` blocks — these are owned exclusively by command wrappers
- `implement-rerun-guard-enhancement`: The implement instruction includes a Step 1b guard that detects fully-applied steps in an existing `implementation.md` and collapses them to `*(already applied)*`
- `scope-reference-consistency`: All scope reminders in instruction files reference `sai-4-apply` (not the legacy `ai-3-apply`)

### Modified Capabilities

- `backfill-spec-path`: Backfill now writes capability specs to `openspec/changes/{name}/specs/{capability}/spec.md` instead of `openspec/specs/{capability}/spec.md`
- `budget-skill-naming`: Budget skill `name:` frontmatter fields use platform-agnostic slugs (`budget-executor`, `budget-explorer`) instead of platform-prefixed names

## Impact

- **New files**: none (new capabilities added to existing instruction/command files)
- **Modified files**: 24 files across `sai/commands/`, `sai/instructions/`, `skills/`, and documentation
- **Out of scope**: `design.md`, `tasks.md`, `implementation.md` — not generated by `/sai-backfill`
