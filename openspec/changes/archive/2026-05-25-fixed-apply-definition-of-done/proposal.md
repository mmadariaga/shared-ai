## Why

The `/sai-4-apply` completion message was ambiguous about when work is truly complete — it only said "once the implementation is done" without accounting for human verification gate reviews and commits. Additionally, the implement instruction's Step 1b (rerun guard) did not explicitly require waiting for the subagent to finish before continuing, risking race conditions in subagent execution.

## What Changes

- **`sai/commands/sai-4-apply.md`**: Updated completion stop condition to clarify that work is complete only after implementation is done, all human verification gates have been reviewed, and commits are done.
- **`sai/instructions/implement.md`**: Added explicit subagent wait instruction to Step 1b — "Wait for the subagent to finish before continuing with any other task."

- `implement-subagent-sync`: Implement rerun guard step explicitly waits for subagent completion before proceeding.
