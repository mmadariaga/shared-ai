## Why

The apply workflow lacked explicit handling for Human Verification steps: the agent could skip presenting the checklist to the user or mark items done without user confirmation. Additionally, no final sweep existed to catch any missed checkboxes before declaring the implementation complete.

## What Changes

- `sai/instructions/apply.md`: Added final sweep directive and explicit Human Verification gate with updated step numbering in the example workflow.

## Capabilities

### New Capabilities

- `apply-human-verification-gate`: When a step contains Human Verification checks, the agent presents the checklist to the user and waits for explicit confirmation before marking those items `[x]` and proceeding to commit.
- `apply-final-sweep`: After all steps complete, the agent scans the entire `implementation.md` and reports any unchecked items before declaring the implementation done.

### Modified Capabilities

- `apply-example-workflow`: Example workflow step numbering updated from 4–6 to 4–8 to reflect the new Human Verification gate inserted between deviation logging and commit proposal.

- **Modified files**: `sai/instructions/apply.md`
- **New files**: none
- **Out of scope**: `design.md`, `tasks.md`, `implementation.md` — not generated by `/sai-backfill`
