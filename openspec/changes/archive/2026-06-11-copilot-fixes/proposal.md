> **⚠ POST-HOC RECORD** — This proposal was backfilled after implementation. It describes a decision already made, not one being proposed.

## Why

GitHub Copilot harness exhibited several issues: agents searched for OpenSpec artifacts with slow recursive globs instead of direct workspace-root paths, the installer only copied flat `sai/instructions/*.md` files so nested instruction subdirectories were silently skipped, the spec phase model assignment underperformed, and the budget-executor agent lacked a directory-listing tool.

## What Changes

- New harness instruction file `sai/instructions/harness/copilot.md` with "Copilot OpenSpec Path Rules": prefer direct `/openspec/...` workspace-root paths, never recursive `**/openspec/...` globs.
- All 14 `commands/copilot/sai-*.prompt.md` wrappers fetch `@sai/instructions/harness/copilot.md` before fetching their command body.
- `commands/copilot/sai-1-spec.prompt.md` model changed from `Claude Sonnet 4.6 (copilot)` to `GPT-5.4 (copilot)`; README model table updated to match for both Copilot columns.
- `bin/install-flow.js`: new exported `listMdFilesRecursive()`; `installClaude`, `installCopilot`, and `installOpencode` now copy `sai/instructions/` recursively, preserving subdirectory structure.
- `INSTALL.copilot.md`: bash and PowerShell manual-install snippets changed to recursive instruction copy.
- `agents/copilot/budget-executor.agent.md`: added `search/listDirectory` tool.
- `agents/copilot/budget-explorer.agent.md`: tool list reordered (no behavioral change).
- `README.md`: Copilot support paragraph now explains *why* the agents window and Copilot CLI are unsupported (prompt files aren't supported there yet).
- Tests added for `listMdFilesRecursive` and for nested instruction copying in the Claude and Opencode install flows.

## Capabilities

### New Capabilities

- `copilot-harness-instructions`: a Copilot-specific harness instruction file defines OpenSpec path-resolution rules and is loaded by every Copilot prompt wrapper.
- `install-instructions-recursive`: all three installers copy `sai/instructions/` recursively, preserving nested subdirectories.
- `spec-copilot-model`: the sai-1-spec Copilot wrapper runs on GPT-5.4.
- `budget-executor-file-discovery`: the Copilot budget-executor agent can list directories via `search/listDirectory`.

### Modified Capabilities

None — no existing capability spec governs the behaviors changed by this diff (verified in conflict detection).
