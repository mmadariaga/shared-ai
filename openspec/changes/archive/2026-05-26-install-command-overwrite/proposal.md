## Why

On reinstall, the install script used `copySkipIfExists` for command wrapper files, preserving stale user-modified copies instead of updating them to the latest versions from the repo. Users who expected reinstalls to refresh their commands got silently outdated wrappers.

## What Changes

- `bin/install-flow.js`: Both `installClaude()` and `installOpencode()` now use `copyWithWarn` instead of `copySkipIfExists` for command wrapper files, so reinstalls overwrite old wrappers while printing a warning.
- `bin/install-flow.js`: Universal skill copies (`budget/SKILL.md`, `sai-commands/SKILL.md`) are repositioned earlier in the install sequence (before `caveman/SKILL.md`) in both functions for consistent ordering.
- `INSTALL.opencode.md`: Post-install section updated to recommend per-project command overrides via `.opencode/commands/` instead of editing global commands directly, with clear copy/edit instructions and a warning about global edits being overwritten on updates.

## Capabilities

### New Capabilities

- `install-command-overwrite`: Install script overwrites existing command wrappers on reinstall with a warning instead of silently preserving stale copies.
- `install-per-project-override-docs`: Installation documentation guides users to copy commands to project-local `.opencode/commands/` for safe customization that survives reinstalls.

- **New files**: none
- **Modified files**: `bin/install-flow.js`, `INSTALL.opencode.md`
- **Out of scope**: `design.md`, `tasks.md`, `implementation.md` — not generated by `/sai-backfill`
