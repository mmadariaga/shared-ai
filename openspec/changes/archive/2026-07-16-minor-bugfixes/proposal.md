## Why

Fixes some minor issues

## What Changes

- `commands/opencode/sai-2-design.md`: replaced the two-line block `**Change-name argument:** $ARGUMENTS` plus `<!-- Optional: append --fast-track after the change name for fast-track mode -->` with a single line `**Change-name argument and and optional flags:** $ARGUMENTS`.
- `commands/opencode/sai-4-apply.md`: same replacement as `sai-2-design.md`.
- `commands/opencode/sai-archive.md`: same replacement as `sai-2-design.md`.
- `sai/instructions/explore.md`: renamed the per-change picker option label `Review sai-2` to `Review sai-2's artifacts` to match the existing `Review sai-1's artifacts` label, and updated three prose references for consistency.

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `opencode-change-arg-passthrough`: the wrapper-echo literal is no longer constrained to the exact `**Change-name argument:**` form; an additional `**Change-name argument and and optional flags:** $ARGUMENTS` form is now also a valid opencode wrapper-echo line, alongside the original form (which the six other opencode change-consuming wrappers still emit).
- `change-picker`: the wrapper-echo scan now also matches the `**Change-name argument and and optional flags:**` form, so the picker resolves the change name from either wrapper-echo variant without falling through to the slower `$ARGUMENTS` / 0-1-N path.
- `sai-fast-track-flag`: the opencode `sai-archive` wrapper no longer carries the HTML-comment consistency marker on its echo line; the echo-line shape itself is the wrapper's mirror-discipline form, and the HTML comment is no longer required.

## Additional Notes

- The wrapper-echo rename in three opencode wrappers is preserved as a second accepted form rather than a replacement, so the six opencode wrappers that still emit the original literal (`sai-3-implement`, `sai-5-review`, `sai-6-security`, `sai-7-performance`, `sai-8-accessibility`, `sai-pr`) continue to function under both the change-picker and the opencode change-arg-passthrough spec.

- The `sai/instructions/explore.md` rename brings the per-change picker label in line with the `explore-post-crystallization-review-loop` spec, which already uses `Review sai-2's artifacts` throughout; no spec change is needed for that capability.
