## Why

`/sai-3-implement` re-explores the codebase to recover stack, conventions, and documentation references that `/sai-2-design` already established, wasting tokens and introducing inference drift. Capturing this information directly in `tasks.md` eliminates the redundant pass and gives the implementation generator authoritative, design-time signal.

## What Changes

- `tasks.md` gains two mandatory sections: `## Required Documentation` and `## Implementation Context`, both generated by `/sai-2-design`
- The `tasks` artifact instruction in `openspec/schemas/sai-workflow/schema.yaml` is updated to require these sections
- `openspec/schemas/sai-workflow/templates/tasks.md` is updated to include the two sections
- `claude/commands/sai-2-design.md` is updated to explicitly populate both sections from codebase research findings
- `claude/commands/sai-3-implement.md` is updated to read these sections directly instead of re-exploring
- `opencode/commands/sai-2-design.md` and `opencode/commands/sai-3-implement.md` receive the same updates

## Capabilities

### New Capabilities

- `tasks-required-documentation`: Mandatory `## Required Documentation` section in `tasks.md` with `### Local files` and `### External URLs` subsections. If nothing to list, agent writes `None`. Schema instruction directs `/sai-2-design` to populate it from docs visited during codebase research.
- `tasks-implementation-context`: Mandatory `## Implementation Context` section in `tasks.md` with three compact fields: **Stack** (primary tech + key versions), **Conventions** (2ŌĆō5 project-specific, non-obvious bullets), **Avoid** (anti-patterns the agent might default to). Schema instruction directs `/sai-2-design` to derive these from codebase research, not copy generic content. Replaces the verbose Expertise Profile template in `plan.md` for sai-3-implement's input contract.

### Modified Capabilities

- `tasks`: existing planning scaffold gains two mandatory trailing sections; step structure (## Step N, Files Affected, What Will Be Done, Testing Strategy) is unchanged.
