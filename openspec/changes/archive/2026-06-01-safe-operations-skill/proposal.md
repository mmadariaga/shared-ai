## Why

Agents performing operations on shared systems or executing destructive commands (git push --force, rm -rf, branch deletion) need explicit guardrails to prevent accidental data loss or irreversible changes. The safe-operations skill enforces reversibility awareness and requires user confirmation before hard-to-reverse or shared-system operations.

## What Changes

- **New skill**: `skills/universal/safe-operations/SKILL.md` — universal skill defining reversibility and impact awareness rules
- **Install scripts**: `INSTALL.claude.md`, `INSTALL.opencode.md`, `bin/install-flow.js` — added safe-operations to the universal skill install sequence
- **Sai command wrappers**: 7 `sai/commands/sai-*.md` body files — added `Fetch @skills/safe-operations/SKILL.md and use it` to load the skill

## Capabilities

### New Capabilities

- `safe-operations-skill`: A universal skill that requires agents to consider reversibility and ask for confirmation before destructive, hard-to-reverse, or shared-system operations
- `install-safe-operations`: Installation scripts copy the safe-operations skill to user-level directories alongside other universal skills

### Modified Capabilities

None — all existing commands and skills remain unchanged; safe-operations is loaded in addition to existing behaviors.
