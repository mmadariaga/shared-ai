> **⚠ POST-HOC RECORD** — This proposal was backfilled after implementation. It describes a decision already made, not one being proposed.

## Why

The LLM was skipping sai-* command file loading and interpreting tasks freely, leading to inconsistent execution of the SAI workflow quality layers (prerequisite checks, caveman mode, cost discipline, phase-specific instructions). A command registry skill was needed to enforce fetch-before-execute discipline for all `/sai-*` commands.

## What Changes

- **New file**: `skills/universal/sai-commands/SKILL.md` — universal skill listing all `/sai-*` commands with their `@commands/` fetch paths and descriptions
- **Modified**: `AGENTS.md` — added documentation entry for the new `sai-commands` skill
- **Modified**: `INSTALL.claude.md` — added copy steps (bash + PowerShell) to install the `sai-commands` skill to `~/.claude/skills/sai-commands/`
- **Modified**: `INSTALL.opencode.md` — added copy steps (bash + PowerShell) to install the `sai-commands` skill to `~/.config/opencode/skills/sai-commands/`

## Capabilities

### New Capabilities

- `sai-command-registry`: A universal skill that lists all `/sai-*` commands with their corresponding `@commands/sai-<name>.md` fetch paths, preventing the LLM from skipping command file loading.
- `fetch-before-execute-enforcement`: The skill mandates that the LLM MUST resolve `/sai-*` commands by reading the corresponding command file before executing any sai-* task, preventing free interpretation.

### Modified Capabilities

- `install-documentation`: Both `INSTALL.claude.md` and `INSTALL.opencode.md` now include installation steps for the `sai-commands` universal skill, following the same pattern as existing skill copy steps.

## Impact

- **New files**: `skills/universal/sai-commands/SKILL.md`
- **Modified files**: `AGENTS.md`, `INSTALL.claude.md`, `INSTALL.opencode.md`
- **Out of scope**: `design.md`, `tasks.md`, `implementation.md` — not generated by `/sai-backfill`
