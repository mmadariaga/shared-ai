> **⚠ POST-HOC RECORD** — This proposal was backfilled after implementation. It describes a decision already made, not one being proposed.

## Why

The Copilot budget subagents (executor, explorer, subagent) had permission issues because they were marked `user-invocable: false`, which prevented Copilot from allowing them to be selected. Additionally, SAI commands and instructions were being installed under the VS Code prompts folder (`prompts/sai/`), conflating prompt files with SAI infrastructure; they now live in a dedicated sibling directory (`Code/User/sai/`).

## What Changes

- `INSTALL.copilot.md` — updated manual install table and bash/PowerShell scripts to use `Code/User/sai/` for SAI commands and instructions
- `agents/copilot/budget-executor.agent.md` — `user-invocable: false` → `true`; description cleaned of "not intended for direct user selection"
- `agents/copilot/budget-explorer.agent.md` — same
- `agents/copilot/budget-subagent.agent.md` — same
- `bin/install-flow.js` — added `getCopilotSaiDir()`; `installCopilot()` now copies SAI files to `saiPath` (`Code/User/sai/`) instead of `promptsPath/sai/`
- `skills/copilot/fetch/SKILL.md` — fetch resolution for `@sai/<subpath>` updated to point to `Code/User/sai/` instead of `prompts/sai/`

## Capabilities

### New Capabilities

- `copilot-sai-folder-path`: SAI commands and instructions are installed to and resolved from a dedicated `Code/User/sai/` directory, separate from the VS Code prompts folder.

### Modified Capabilities

- `copilot-budget-agents-user-invocable`: Budget agent descriptors (executor, explorer, subagent) now allow direct user selection in Copilot (`user-invocable: true`).

## Impact

- **New files**: none
- **Modified files**: `INSTALL.copilot.md`, `agents/copilot/budget-executor.agent.md`, `agents/copilot/budget-explorer.agent.md`, `agents/copilot/budget-subagent.agent.md`, `bin/install-flow.js`, `skills/copilot/fetch/SKILL.md`
- **Out of scope**: `design.md`, `tasks.md`, `implementation.md` — not generated by `/sai-backfill`
