## Why

Runtime path resolution errors in OpenCode when using `Fetch @` directives. The fetch skill attempted to read files directly without first checking if they existed, causing failures when project-local files were absent.

## What Changes

- **`skills/claude/fetch/SKILL.md`**: Updated fetch resolution to use `glob/LS` to check file existence before `Read`, instead of attempting `Read` directly and relying on error handling.
- **`skills/opencode/fetch/SKILL.md`**: Same change — use `glob/LS` existence check before `Read` for project-local paths.
- **`commands/opencode/sai-explore.md`**: Fixed model reference from `opencode/qwen3.6-plus` to `opencode-go/qwen3.6-plus`.
- **`sai/instructions/prereqs.md`**: Added explicit verification command (`openspec --version`) for the openspec binary PATH check.

## Capabilities

### New Capabilities

- `fetch-existence-check-before-read`: Fetch skill uses glob/LS to verify file existence before attempting Read, preventing runtime errors when project-local paths are absent.

### Modified Capabilities

- `opencode-model-resolution`: Model reference in sai-explore command corrected to use proper namespace `opencode-go/qwen3.6-plus`.
- `prereq-verification`: Prerequisite check instructions now include explicit verification command for cross-platform compatibility.
