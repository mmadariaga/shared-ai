> **⚠ POST-HOC RECORD** — This proposal was backfilled after implementation. It describes a decision already made, not one being proposed.

## Why

Opencode runtime issues caused incorrect behavior in the backfill interview flow and missing compatibility metadata on Claude-only skills. The backfill command asked both interview questions in a single message instead of sequentially, and Claude-specific skills lacked explicit `compatibility: claude` declarations and "NOT compatible with opencode" clarifications in their descriptions.

## What Changes

- `sai/instructions/backfill.md` — Phase 2 interview questions now asked one at a time, waiting for user response between each, instead of batched in a single message
- `skills/claude/budget-executor/SKILL.md` — added "Claude Code only — NOT compatible with opencode." to description and `compatibility: claude` frontmatter field
- `skills/claude/budget-explorer/SKILL.md` — same compatibility clarification and frontmatter field
- `skills/claude/fetch/SKILL.md` — clarified "Claude Code only — NOT compatible with opencode." in description

## Capabilities

### New Capabilities

- `backfill-sequential-interview`: Interview questions in Phase 2 are asked one at a time, with the agent waiting for the user's full response before proceeding to the next question.
- `claude-skill-compatibility-metadata`: Claude-only skills carry explicit `compatibility: claude` frontmatter and "NOT compatible with opencode" text in their description to prevent cross-platform misuse.

### Modified Capabilities

- `backfill-interview`: Extended from "two fixed questions always asked" to include sequential delivery constraint — questions asked individually, not merged in one message.

## Impact

- **New files**: none
- **Modified files**: `sai/instructions/backfill.md`, `skills/claude/budget-executor/SKILL.md`, `skills/claude/budget-explorer/SKILL.md`, `skills/claude/fetch/SKILL.md`
- **Out of scope**: `design.md`, `tasks.md`, `implementation.md` — not generated by `/sai-backfill`
