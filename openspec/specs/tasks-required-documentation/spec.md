## Requirements
### Requirement: tasks-required-documentation-section
`tasks.md` SHALL contain a `## Required Documentation` section generated by `/sai-2-design` immediately after all implementation steps.

The section SHALL have two subsections:

    ### Local files
    <!-- one path per line; line ranges where only a portion applies -->

    ### External URLs
    <!-- one URL per line -->

If no local files apply, the agent SHALL write `None` under `### Local files`. If no external URLs apply, the agent SHALL write `None` under `### External URLs`.

The section SHALL NOT be left empty or omitted.

#### Scenario: sai-2-design populates both subsections from research findings

- **WHEN** `/sai-2-design` generates `tasks.md` after researching the codebase and external docs
- **THEN** `tasks.md` contains `## Required Documentation` with `### Local files` listing every file read during design-phase research and `### External URLs` listing every external doc consulted; any subsection with nothing to list contains only `None`

#### Scenario: sai-3-implement reads section without re-exploring

- **WHEN** `/sai-3-implement` reads `tasks.md` to produce `implementation.md`
- **THEN** it reads only the files and URLs listed under `## Required Documentation` and does not perform additional codebase exploration to recover this information

#### Scenario: no relevant external URLs

- **WHEN** `/sai-2-design` finds no external documentation relevant to the change
- **THEN** `tasks.md` contains `### External URLs` followed by `None`

### Requirement: tasks-required-documentation-schema
The `tasks` artifact instruction in `openspec/schemas/sai-workflow/schema.yaml` SHALL include a directive requiring `/sai-2-design` to populate `## Required Documentation` from docs visited during codebase research.

The instruction SHALL specify that specific sub-files or sections with line ranges are preferred over entire file paths when only a portion applies.

#### Scenario: schema instruction includes required documentation directive

- **WHEN** `openspec instructions tasks --change <name>` is run
- **THEN** the returned `instruction` field references the `## Required Documentation` section and explains that it must be populated from design-phase research findings, not left empty
