# proposal-research-documentation Specification

## ADDED Requirements

### Requirement: proposal.md SHALL include a Proposal Research Documentation section

Every `proposal.md` artifact generated by `/sai-1-spec` SHALL contain a `## Proposal Research Documentation` section with two subsections: `### Local files` and `### External URLs`. Each subsection lists the files or URLs consulted during spec-phase codebase research. If none apply, the subsection contains `None`.

#### Scenario: sai-1-spec populates research documentation from spec-phase research

- **WHEN** `/sai-1-spec` writes `proposal.md` after researching the codebase and external docs
- **THEN** `proposal.md` contains `## Proposal Research Documentation` with `### Local files` listing every local path read during research and `### External URLs` listing every external URL consulted; any subsection with nothing to list contains only `None`

#### Scenario: no external URLs consulted

- **WHEN** `/sai-1-spec` researches only local files with no external documentation
- **THEN** `proposal.md` contains `### External URLs` followed by `None`

### Requirement: sai-workflow schema declares Proposal Research Documentation field for the proposal artifact

The proposal artifact instruction in `openspec/schemas/sai-workflow/schema.yaml` SHALL include a `**Proposal Research Documentation**` directive requiring agents to list files and URLs consulted during spec-phase research.

#### Scenario: schema instruction references research documentation

- **WHEN** `openspec/schemas/sai-workflow/schema.yaml` is read
- **THEN** the proposal artifact description contains a `**Proposal Research Documentation**` bullet describing what to record

### Requirement: proposal.md template includes Proposal Research Documentation scaffold

`openspec/schemas/sai-workflow/templates/proposal.md` SHALL contain a `## Proposal Research Documentation` section with `### Local files` and `### External URLs` comment placeholders.

#### Scenario: new proposal created from template includes scaffold

- **WHEN** a new proposal.md is created from the sai-workflow template
- **THEN** it contains `## Proposal Research Documentation` with subsections ready to be filled in
